## Переменные и их значения
- В Python нет объявления переменных, официально это называется assignment — «назначение, связывание». Мы связываем имя и значение.
- При именовании переменных принято использовать snake_case.
- Что есть литерал?
```python
# Прямо в коде явно указано значение переменной - этой переменной значение
# задано при помощи литерала. В данном случае 15 - числовой литерал.
quantity_of_jars = 15

# А значение этой переменной вычисляется в ходе арифметической операции -
# значение переменной указано неявно, через выражение.
total_quantity = quantity_of_jars + 10

print(total_quantity)
```
## Типизация и типы данных
- Python относится к языкам с **динамической** и **сильной типизацией**.
- Узнать тип переменной можно с помощью функции `type()`.
- В языке Python — сильная типизация. Python не станет сам преобразовывать один тип данных в другой ради того, чтобы выполнить какие-то операции. При необходимости разработчик должен сам преобразовать один тип в другой.
```python
# Тут речь идет про знакомую нам со школы штуку
name = 'Frol'
age = 15
print(name + ' ' + str(age))
```

- Параметры — это названия переменных, которые указывают в круглых скобках при объявлении функции. Аргументы — это значения параметров. Их передают в функцию, указывая в круглых скобках при вызове функции.
```python
# Теперь у функции welcome() два параметра: profession и name.
def welcome(profession, name):
    # Оба параметра применяются в теле функции.
    print('Добро пожаловать в программу,', profession, name)

# Первый вызов функции с одними аргументами.
welcome('фермер', 'Тоня')

# Второй вызов функции, с другими аргументами.
welcome('садовод', 'Женя')
```
```python
# Если при вызове забудут передать имя, значением name будет слово 'Пользователь';
# а если не будет передано название продукта,
# функция присвоит переменной product значение 'разные продукты'.
def action(name='Пользователь', product='разные продукты'):
    print(name, 'заготавливает на зиму', product)

# Передаётся только один аргумент из двух.
action('Женя')
# А можно вызвать функцию вообще без аргументов - 
# и оба параметра примут значения по умолчанию.
action()
```
- Оба параметра в этой функции передавать необязательно, поэтому такие параметры называют необязательными или опциональными. При объявлении функции можно комбинировать обязательные параметры с опциональными.
```python
def action(name, food):
    print(name, 'заготавливает на зиму', food)

action(food='инжир', name='Женя')
```
- После того как вы передали именованный аргумент в функцию, вы уже не можете передавать позиционные аргументы.
```python
def return_literal():
    return 'Литерал - это явно указанное конкретное значение.'

def return_variable(value):
    result = value * 2
    return result  # Возвращаем значение переменной.

def return_expression(value):
    return value * 3  # Возвращаем результат вычисления выражения.

result_return_literal = return_literal()  # Вызов.
result_return_variable = return_variable('100')  # Вызов.
result_return_expression = return_expression(100)  # Вызов.
```
- У инструкции `return` есть одно важное свойство: она не только возвращает результат, но и прерывает работу функции. Код, который следует за `return`, никогда не будет выполнен. В функции может быть несколько инструкций `return`, но выполнится только одна из них — та, которая в ходе выполнения программы будет обработана первой. После этого функция прекратит работу, а оставшуюся часть кода функции Python проигнорирует.
```
def empty():
    pass

empty()
```
- Чтобы создать пустую функцию и вместе с тем избежать ошибок, в тело пустой функции ставят «заглушку» — инструкцию `pass`. Инструкция `pass` — это оператор-заполнитель, который ничего не делает и используется для поддержания целостности структуры программы.
- Есть и другой вариант: вместо инструкции pass в тело пустой функции можно поставить значение **Ellipsis** (с англ. «троеточие» или «многоточие»), оно записывается как три точки: `...`.
